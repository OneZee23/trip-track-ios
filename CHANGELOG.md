## [0.4.3] -- 18 апреля 2026

### Исправлено
- **Ложное автозавершение поездки в auto-режиме** — CMMotion давал событие `automotive → not automotive` на светофорах и в пробках, после чего через ~4 минуты поездка автоматически прерывалась, хотя машина продолжала ехать. В длинных поездках одна запись разбивалась на 5+ сегментов. Motion-ended больше не триггерит auto-stop — эта ответственность остаётся только за GPS-скоростью (`<2 км/ч` непрерывно 20 минут) и реальным отключением Bluetooth
- **Pending-таймер авто-стопа отменяется, если машина снова поехала** — после prompt "Машина не движется" 3-минутный таймер автозавершения теперь сбрасывается при возврате активности `automotive`

### Изменено
- **Тексты уведомлений об автозавершении** — больше не пишем "Bluetooth отключился" для всех случаев. BT-disconnect → "Bluetooth отключился. Автозавершение через N мин"; 20-мин неподвижность → "Машина не движется. Автозавершение через N мин"
- **Определение новой поездки в remind-режиме** — флаг "уже показывал reminder" сбрасывается при 10+ минутах неактивности, реальном BT-disconnect или автостопе. Раньше сброс был только по 60-сек debounce motion-ended, который больше не используется
- Версия 0.4.2 → 0.4.3, build 8 → 9

## [0.4.0] -- 15 апреля 2026

### Добавлено
- **CoreData Model Versioning** -- модель данных версионирована (v1 → v2). Все будущие миграции схемы безопасны для существующих пользователей (lightweight migration)
- **Подготовка к серверному синку** -- новые поля на сущностях: `userId`, `serverCreatedAt`, `conflictVersion` (TripEntity), `remoteURL`, `uploadStatus` (TripPhotoEntity), `userId` (VehicleEntity)
- **Codable на моделях** -- `Trip`, `TrackPoint`, `TripPhoto`, `Vehicle` сериализуются в JSON для будущего API
- **TripRepository** -- protocol-абстракция над CoreData CRUD. TripManager делегирует хранение в repository, что позволит подключить серверный синк без переписывания ViewModel'ей
- **User Identity** -- локальный `userId` (UUID) генерируется при первом запуске и проставляется на все существующие сущности. Фундамент для привязки данных к аккаунту
- **Photo Sync Readiness** -- `PhotoUploadStatus` enum (localOnly/uploading/uploaded/failed), `RemotePhotoStorage` protocol (stub), методы `pendingUploads()` и `markUploaded()` в PhotoStorageService
- **SyncQueue** -- очередь синк-операций с дедупликацией, приоритизацией (metadata → photos), exponential backoff retry, интеграция с NetworkMonitor
- **Тесты** -- `CodableRoundTripTests` (6 тестов на сериализацию), `SyncQueueTests` (6 тестов на очередь)

### Изменено
- TripManager CRUD-методы делегируют в TripRepository (−280 строк)
- `thumbnailDiskURL` теперь проверяет path traversal (как `safePhotoURL`)
- SyncQueue помечен `@MainActor` для thread safety
- Все CoreData fetch-запросы имеют `fetchBatchSize` для оптимизации памяти
- `PhotoStorageService.markUploaded` использует `PersistenceController.save()` вместо raw `context.save()`
- Версия 0.3.1 → 0.4.0, build 5 → 6

## [0.3.1] -- 13 апреля 2026

### Добавлено
- **Онбординг автозаписи** -- 4-я страница при первом запуске: объяснение зачем нужны фоновая геолокация и датчик движения, кнопка «Включить» запрашивает все разрешения и активирует режим «Напоминание». Можно пропустить и настроить позже
- **Детекция вождения через CMMotion** -- `CMMotionActivityManager` определяет `.automotive` activity в реальном времени (foreground) и ретроспективно (background wake)
- **GPS watchdog** -- если за 60 секунд не приходит ни одного валидного GPS-обновления, трекинг автоматически перезапускается
- **GPS warm-up** -- первые 2 секунды после переключения в режим записи данные игнорируются (устранение скачков скорости)
- **Автозавершение по неподвижности** -- если скорость < 2 км/ч непрерывно 20 минут, предложение завершить поездку
- **Восстановление начала поездки** -- при автостарте через CMMotion запрашиваем реальное время начала `.automotive` activity и бэкдэйтим `startDate` поездки
- **Background task** -- `beginBackgroundTask` при автостарте из фона, чтобы iOS не усыпил приложение пока GPS прогревается

### Изменено
- GPS точность в режиме записи: `kCLLocationAccuracyBest` → `kCLLocationAccuracyBestForNavigation` (добавляет акселерометр + гироскоп)
- Строгая фильтрация GPS: отклонение `speed < 0`, `accuracy > 50м`, stale > 10с при записи
- Notification categories перерегистрируются при смене языка приложения
- `LocationManager.startTracking()` идемпотентен (повторный вызов безопасен)
- Версия 0.3.0 → 0.3.1, build 4 → 5

### Исправлено
- Скачки скорости (0, 50, 20, 33 км/ч) в первые 10 секунд после автостарта записи
- «Мёртвая запись» (0 км после 10+ км) при автостарте из фона и блокировке экрана
- `locationManagerDidPauseLocationUpdates` принудительно возобновляет GPS если идёт запись

## [0.3.0] -- 10 апреля 2026

### Добавлено
- **Автозапись поездок по Bluetooth** -- привязка BT-устройства (магнитолы) к машине в деталке автомобиля. Два режима: "Напоминание" (уведомление с кнопкой "Начать запись") и "Авто" (запись стартует автоматически)
- **Детекция вождения через CMMotion** -- `CMMotionActivityManager` определяет `.automotive` activity. Работает после force-quit через `significantLocationChanges` (требует Always location)
- **Трёхслойная детекция**: (1) Audio route -- мгновенная при подключении к BT, app в памяти; (2) CMMotion + significant location -- после force-quit, задержка 1-3 мин; (3) Проверка при разблокировке через `didBecomeActive`
- **Привязка BT-устройства к конкретной машине** -- при подключении к магнитоле автоматически выбирается нужный vehicle
- **Несколько BT-устройств на машину** -- можно привязать и магнитолу, и другие устройства
- **BLE-сканер устройств** (`BluetoothScanSheet`) -- поиск Bluetooth-устройств поблизости с индикатором силы сигнала (цветные бары), сортировка A-Z
- **Уведомления с action buttons** -- "Начать запись" / "Пропустить" на lock screen. Тап по уведомлению = старт записи + переход на экран Record
- **Автозавершение поездки** -- при отключении BT или окончании `.automotive` activity, таймер автозавершения (1-10 мин, настраивается). Debounce 60 секунд для CMMotion (светофоры, пробки)
- **Карточка Bluetooth в деталке машины** -- toggle включения, описание режимов, список привязанных устройств, настройка таймаута автозавершения
- **NotificationManager** -- локальные уведомления с категориями и action buttons
- **AudioRouteDetector** -- мониторинг `AVAudioSession.routeChangeNotification` для Classic Bluetooth (A2DP/HFP)
- **BluetoothDetector** -- `CBCentralManager` с state restoration для BLE-устройств
- **MotionDetector** -- обёртка над `CMMotionActivityManager` с live updates и historical query
- Новые permissions: `NSBluetoothAlwaysUsageDescription`, `NSMotionUsageDescription`, `bluetooth-central` background mode

### Изменено
- `MapViewModel.startRecording()` / `stopRecording()` стали public для вызова из AutoTripService
- Версия 0.2.0 → 0.3.0, build 3 → 4
- `LanguageManager` -- добавлен `static var currentLanguage` для использования вне SwiftUI views
- `SettingsManager` -- добавлены `autoRecordMode`, `autoStopTimeout`, `savedBluetoothDevices` (UserDefaults)

## [0.2.0] -- 9 апреля 2026

### Добавлено

- **Temporal fog** -- на экране деталки поездки туман показывает состояние на момент завершения поездки (`visitedHashes(before: trip.endDate)`)
- **Экран детальной информации о машине** (`VehicleDetailView`) -- тап на карточку машины в профиле: большой аватар, редактирование имени, одометр с прогресс-баром до следующего уровня, сетка всех 10 стикеров (полученные/заблокированные), выбор аватара (pixel cars + emoji), настройки расхода топлива, единицы измерения, дата создания
- Смена аватара машины (`SettingsManager.updateVehicleAvatar`)
- Кнопка "Подробнее" в меню три точек в гараже -- переход в деталку любой машины
- Тап на активную машину в гараже открывает её деталку
- Лимит 5 машин в гараже с подписью "Максимум 5 автомобилей"
- Сортировка машин в гараже по пробегу (убывание)
- Адаптивный зум на карте регионов -- ближе при малом количестве тайлов, дальше при большом (`min(150km, max(5km, tiles * 350))`)
- Тёмный fog-фон при overscroll на экране деталки поездки

### Изменено
- Настройки расхода топлива и единиц измерения перенесены из гаража в деталку машины
- Рендерер тумана использует пространственное индексирование по geohash4 префиксам для эффективного поиска видимых центров
- Убран компас с карт деталки поездки и регионов
- Убран parallax-stretch на карте деталки поездки
- Отключен bounce у ScrollView на экране деталки поездки
- Pull-to-refresh на экране регионов больше не дёргает заголовок (батчинг state updates + задержка dismiss)

### Исправлено
- Туман не пропадает при старте/остановке записи
- Туман корректно пересобирается после завершения поездки с учётом новых тайлов
- Потокобезопасность: анимация тумана на главном потоке, рендеринг на фоновом потоке MapKit -- данные изолированы через prefix index snapshot

### Удалено
- Старый `FogPolygon` / `FogMaskGenerator` / `FogOverlayRenderer` из `Views/Regions/` -- заменены единым `FogPolygonBuilder.swift`

## [0.1.0] -- 24 марта 2026

Первый релиз. Полнофункциональный автодневник поездок.

### Добавлено

**Запись маршрута**
- GPS-трекинг с фильтром Калмана (`SmoothTrackManager`) для сглаживания координат и удаления шума
- Фоновая запись маршрута (`UIBackgroundModes: location`)
- Реалтайм-показатели во время записи: текущая скорость, высота, пройденная дистанция, время
- Сворачиваемая панель метрик (compact/expanded `TrackingHUD`) -- тап переключает между режимами
- Компактный HUD (`CompactTrackingHUD`) с минимальной информацией
- Спидометр (`SpeedometerView`) с визуальной шкалой
- Индикатор качества GPS-сигнала (`GPSIndicatorView`)
- Плавная анимация "головы" маршрута с glow-эффектом (`GlowingHeadOverlay`)
- Автоматический zoom карты по скорости движения (ближе на малых скоростях, дальше на высоких)
- Пауза и возобновление записи
- Автоматическая темная/светлая тема карты по времени суток (`SunCalculator` -- расчет sunrise/sunset по координатам)
- Idle-экран с пульсирующей кнопкой старта, суммарным пробегом и количеством поездок
- Экран итогов после завершения поездки (`TripCompleteSummaryView`): карта маршрута, статистика, заработанные XP, полученные бейджи, кнопки добавления фото и заметок

**Лента поездок**
- Группировка поездок по месяцам с collapsible-секциями
- Contribution calendar -- GitHub-style heatmap с интенсивностью по километрам за день
- Фильтрация по дате через calendar picker (выбор диапазона)
- Фильтрация по региону (чипы уникальных регионов из геокодинга)
- Поиск по названию поездки
- Быстрая статистика в header: общий пробег, количество поездок, суммарное время
- Пагинация -- 20 поездок на страницу, подгрузка при скролле
- Карточки поездок с превью маршрута (`LightRoutePreview`), названием, датой, дистанцией, временем
- Swipe-удаление с undo-таймером (5 секунд) и toast-уведомлением
- Skeleton-загрузка карточек при первом открытии
- Empty state с предложением начать первую поездку
- Подробная статистика (`StatsView`): текущий/прошлый месяц, недельная активность (Swift Charts), calendar heatmap, топ мест, рекорды

**Детали поездки**
- Интерактивная карта с маршрутом и parallax-stretch при скролле
- Маршрут раскрашен градиентом по скорости (`RouteMapView`)
- Статистика: дистанция, время, средняя/максимальная скорость, набор высоты
- Фотографии поездки -- добавление через `PhotoPickerView`, полноэкранный просмотр (`PhotoFullScreenView`), удаление
- Редактирование названия поездки (inline edit с фокусом)

- Share-карточка (`TripShareCardView`) для Instagram/соцсетей: маршрут + статистика + брендинг
- Добавление текстовых заметок (`NotesEditorView`)
- Бейджи, заработанные за поездку (`TripBadgesRow`)

**Карта территорий**
- Scratch-карта (fog of war) на основе geohash precision-6 (`GeohashEncoder` + `VisitedGeohashEntity`)
- Fog-маска генерируется на GPU (`FogMaskGenerator`, `FogOverlayRenderer`)
- Fullscreen-режим карты (`FullscreenFogMapView`)
- Статистика: количество посещенных тайлов
- Списки посещенных городов и регионов с количеством визитов
- Все маршруты всех поездок отображаются как polyline overlay
- Backfill при первом запуске -- обрабатывает существующие поездки

**Геймификация**
- 30+ бейджей в 4 категориях: дистанция, исследование, особые, серии
- Бейджи дистанции: Первый путь, Завсегдатай (10 поездок), Дорожный воин (50), километровые вехи (1/100/1000/10000 км), Марафонец (42.2 км за поездку), Стоик (100+ км за поездку), Железная задница (500+ км за поездку), Спид-демон (120+ км/ч)
- Скрытые бейджи (`isHidden`) -- не видны до получения
- Повторяемые бейджи (`isRepeatable`) -- можно получить несколько раз
- Celebration-экран с конфетти при получении бейджа (`BadgeCelebrationView`, `ConfettiView`)
- Детальный overlay для каждого бейджа (`BadgeDetailOverlay`)
- Каталог всех бейджей (`BadgesView`) с группировкой по категориям
- XP-система: 1 XP за километр, x2 бонус за поездки 200+ км, +20 XP за первую поездку дня, +50 XP за новый регион, x1.5 за поездку в новом регионе
- Уровни водителя (1-30) с прогрессивной шкалой XP (`LevelSystem`)
- Ранги: Новичок (1-4), Водитель (5-9), Путешественник (10-14), Исследователь (15-19), Навигатор (20-24), Дальнобойщик (25-29), Легенда (30)
- Daily streak -- серия дней с поездками
- Профиль (`ProfileView`) с выбором аватара (8 эмодзи), карточкой уровня (`DriverLevelView`), быстрой статистикой

**Коллекция дорог**
- Автоматическое обнаружение часто проезжаемых дорог (`RoadCollectionManager`)
- Уровни освоения дороги (discovered, familiar, mastered)
- Карточки дорог с информацией о редкости
- Фильтрация по редкости

**Гараж**
- Профиль автомобиля с одометром (`VehicleCardView`)
- Экран управления машинами (`GarageView`)

**Интерфейс**
- Кастомный glass tab bar (`CustomTabBar`) -- 3 вкладки: лента, запись, территории
- Центральная кнопка записи (оранжевая пульсация в idle, красный стоп при записи)
- UI-модификаторы для единообразия карточек: `.surfaceCard()`, `.glassBackground()`, `.glassPill()`
- Адаптивная цветовая палитра (`AppTheme`) для светлой и темной темы
- Toast-уведомления (`ToastView`)
- Баннер записи (`RecordingBanner`) -- видно на всех вкладках
- Haptic feedback на все интерактивные элементы

**Настройки**
- Тема оформления: системная / светлая / темная (`ThemeManager`)
- Язык интерфейса: русский / английский -- переключается на лету без перезапуска (`LanguageManager`)
- Все строки UI через `AppStrings` enum -- нет hardcoded текста
- Отображение версии и номера сборки из bundle

**Онбординг**
- 3 страницы при первом запуске: приветствие, описание записи, запрос разрешения на геолокацию
- Swipe между страницами (TabView с page style)
- Кнопка "Поехали!" на финальной странице запрашивает location permission

**Данные и persistence**
- CoreData с 8 entities: `TripEntity` (центральная), `TrackPointEntity`, `TripPhotoEntity`, `VehicleEntity`, `UserSettingsEntity`, `VisitedGeohashEntity`, `RoadEntity`
- Cascade-удаление: удаление поездки удаляет все track points и фото
- Batch saves: точки GPS батчатся по 10 штук или каждые 15 секунд (`saveBatchSize`, `saveInterval`)
- Асинхронное сохранение (`PersistenceController.saveAsync()`) для неблокирующих записей из location callbacks
- Binary polyline encoding (`Trip.encodePolyline/decodePolyline`) -- компактное хранение маршрута в Float32 парах
- Превью-полилайн для карточек ленты (упрощенный маршрут)
- Автоудаление мусорных поездок: <500 метров AND <2 минуты
- Очистка orphaned trips (без endDate) при старте
- Фото хранятся в Documents directory (`PhotoStorageService`), миниатюры через `AsyncThumbnailView`

**Разработка**
- Provider pattern для location: `LocationProvider` протокол, `RealGPSProvider` (CoreLocation) + `SimulatedLocationProvider` (dev-джойстик)
- Dev mode с виртуальным джойстиком для тестирования без реального GPS
- Dev menu (`DevMenuView`) для переключения режимов
- Мониторинг сети (`NetworkMonitor`)
- Сборка через `xcodegen` (`project.yml`), локальный signing через `Local.xcconfig` (gitignored)
- Нулевые внешние зависимости -- 100% нативные фреймворки Apple
